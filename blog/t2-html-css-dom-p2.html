Grid Based Design

Grid based design is a way of laying out a web page using a grid structure, made up of rows and columns. For responsive design, the most common grid layout utilises 12 columns, and a total width of 100%, and will shrink and expand as you resize the browser window.


Hype about Responsive Pages

In this device-oriented world we live in, web pages need to be both functional and aesthetically pleasing on all sized devies, from mobile phone screens to tablets to desktops. In fact, developers should be designing websites using 'mobile-first' design as mobile-phones overtook desktop computers as the most popular device worldwide for internet usage in 2014, and this figure continues to climb.


Semantic Structure

Semantic elements in HTML are those that have meaning, ie. tell us about the contents of that element. Some common semantc elements are:

  • Headers - h1, h2, h3... h6
  • Images - img
  • Paragraphs - p
  • Forms - form
  • Tables - table
  • Articles - article

Using Semantic Structure in your HTML means using these tags where they should be, to define the structure of those particular parts of your page.


My take on Frameworks and Responsive layouts is that using these allows a developer to see the big picture view of their page, and how all the elements on the page relate to each other. It is a straight-forward, no-mess solution to laying out a web page and ensuring the page will look great no matter what device it is viewed from!

html, css and the dom - part 1

Analogy for HTML, CSS and the DOM

The best analogy I have heard to describe HTML, CSS and the DOM is based on the concept of a house. HTML (HyperText Markup Language) is the house, it has a structure - rooms, windows, stairs etc. CSS (Cascading Style Sheets) is the interior decorating - the colour and fabric of the curtains, the wallpaper or paint and the lightshades. The DOM (Document Object Model) is the blueprint of this house - a visual representation that can be used to shift walls or add extra features to the house.


Boxifying Design

This refers to way in which a web page is constructed from a series of different sized boxes. There are main boxes for major sections of a page - the nav menu, the main content and the footer for example, then smaller boxes inside these boxes for things like headings, images and text. There may even be boxes inside these boxes for individual links, captions or lists.


The Box Model

Within all these boxes mentioned above, you can find a model that conveys the content of the box and how it is laid out - The Box Model. This diagram shows the Box Model in all its glory:


The Box Model is basically a box that wraps around every single HTML element, and is made up of:

  • content
  • padding
  • border
  • margin

All these can be given values to dictate how the box will look.

JavaScript in comparison to HTML and CSS

From a functionality point of view, JavaScript allows us to make static web pages much more interactive, with elements that respond to actions to produce a desired result, such as animated graphics, pop-ups, timed content updates etc. From an implementation point of view, so far JavaScript is a lot more complex and harder to get your head around than HTML or CSS as there are so many different componenets, all with particular syntax.


Control flow and loops

Control flow is the order in which a computer executes statements in a script, which in basic script would run from the first line through to the last line of code. Loops interrupt this flow by introducing conditions which have to be met for the flow to continue. This process could be applied to a real-life situation such as 'brushing your teeth':

Basic script:

  1. Pick up toothbrush
  2. Apply toothpaste
  3. Brush teeth
  4. Rinse
  5. Put toothbrush down

Script with Loops:

  1. Pick up toothbrush
  2. Apply toothpaste
  3. Brush teeth
  4. Rinse
  5. IF teeth feel dirty still
  6. Repeat brushing
  7. ELSE Put toothbrush down


Accessing data from Arrays and Objects

Arrays are used to store a bunch of numbers, or a list of objects of the same type, in a single variable. To access the data from an array you refer to the index number of the item you want ie. if you want the fifth number in an array you would ask for Array[4] (arrays are indexed at 0).

To access the data from an object, which will be a property of that object, you will ask for the object[property] or object.property.


Functions

A JavaScript function is a block of code designed to perform a particular task. A JavaScript function is executed when "something" invokes it (calls it). Functions are the verbs of Javascript - they make actions happen. Without them you would just have a whole bunch of data stored and nothing happening to it.
